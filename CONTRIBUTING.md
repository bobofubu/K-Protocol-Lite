# 贡献指南 Contributing Guide

感谢你有意为 K-Protocol-Lite 做出贡献。请在提交前阅读本文件。

## 贡献原则

本项目核心文件为 `core_directive.md`，任何修改必须满足：

1. **不降低推理质量**：修改后须通过极端发散性测试
2. **不增加 Token 负担**：单次指令 Token 消耗不超过修改前的 120%
3. **不破坏状态机**：自我批评与结构化输出的触发顺序必须保持完整
4. **模型兼容下限**：请在至少一个参数量 ≥ 70B 的模型上验证

## 如何提交优化建议

1. 先开一个 **Issue**，使用「Prompt Optimization Report」模板
2. 等待 maintainer 确认方向后，再提交 PR
3. PR 中必须完成所有 Checklist 项目

## 不接受的提交类型

- 仅修改 README 而无实质协议改进
- 未附对比测试快照的修改
- 在 Qwen-7B 等小模型上测试的结果（不在适用范围内）

## 联系方式

如有疑问，请直接在 Issue 中留言。
